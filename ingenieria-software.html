¿Qué es Ingeniería de Software?

Disciplina para construir software útil, mantenible y seguro con procesos, técnicas y herramientas. No es solo programar: incluye análisis, diseño, validación, despliegue y operación.

🎯 Objetivo de negocio

Entregar valor medible (ingresos, eficiencia, cumplimiento, experiencia de usuario).

⚖️ Restricciones

Tiempo, costo, alcance, riesgos, seguridad, regulación y capacidades del equipo.

🧱 Calidad integral

Funcionalidad, confiabilidad, rendimiento, mantenibilidad, usabilidad y portabilidad.

Las 5 preguntas iniciales

1) ¿Qué problema resolvemos?

Define dolor, impacto y prioridad de negocio.

2) ¿Para quién?

Segmentos de usuario, roles, necesidades y contexto operativo.

3) ¿Cómo medimos éxito?

KPIs: adopción, tiempo de proceso, errores, satisfacción, NPS.

4) ¿Qué riesgos hay?

Dependencias, deuda técnica, seguridad, incertidumbre de alcance.

Formato práctico de historia de usuario

Como analista de compras, quiero aprobar solicitudes desde móvil, para reducir tiempos de ciclo.

  • Criterio 1: Si monto > 5.000, solicitar doble aprobación.
  • Criterio 2: Registrar auditoría de usuario, fecha y motivo.
  • Criterio 3: Respuesta en menos de 2 segundos para 95% de casos.

Preguntas aplicadas (Java / Spring MVC)

Banco de preguntas integrado desde tu carpeta de estudio, con respuesta y explicación corta para repaso.

@Controller + retorno de String en Spring MVC

Pregunta: ¿Qué rol cumple devolver rutas como "categoria/formulario_categoria" o "redirect:/categorias/listar"?

Respuesta: La clase gestiona peticiones web y retorna vistas (o redirecciones) renderizadas por un motor como Thymeleaf, interactuando con el modelo.

Thymeleaf con method="post"

Pregunta: Si el formulario usa th:action="@{/procesar}" method="post", ¿qué método debe manejarlo?

Respuesta: @PostMapping("/procesar") (o @RequestMapping(method = POST) equivalente).

Uso de Optional en repositorios

Pregunta: ¿Por qué usar Optional<Estudiante> antes de eliminar?

Respuesta: Para manejar de forma segura el caso donde no existe el registro y evitar errores por nulos.

Polimorfismo en Java

Pregunta: Si Padre obj = new Hijo(); obj.mostrar();, ¿qué imprime?

Respuesta: "Mostrar en Hijo", por despacho dinámico (método sobrescrito en tiempo de ejecución).

Estrategia de pruebas

La calidad se construye desde el diseño, no se agrega al final.

NivelObjetivoHerramientas típicas
UnitariasValidar lógica aisladaxUnit, Jest, PyTest
IntegraciónValidar interacción entre módulosTestContainers, Postman/Newman
E2EValidar flujo de negocio completoCypress, Playwright, Selenium
No funcionalesCarga, seguridad, resilienciak6, JMeter, OWASP ZAP

Pirámide de pruebas (recomendación)

Base amplia: Unitarias

Rápidas, baratas, muchas. Detectan errores temprano.

Capa media: Integración

Validan contratos entre componentes y servicios externos.

Cima: E2E

Pocas, críticas y estables. Cubren los flujos de mayor valor.

Regla práctica

Más pruebas automáticas en lógica de negocio; menos en UI frágil.

Gestión de proyectos: PMI, PMP y PMBOK

PMI

Project Management Institute: organismo que define estándares globales de dirección de proyectos.

PMP

Project Management Professional: certificación de gestión de proyectos basada en buenas prácticas del PMI.

PMBOK

Guía de fundamentos de dirección de proyectos. Estructura procesos, dominios, principios y áreas de conocimiento.

Aplicación en software

Se usa para plan de alcance, cronograma, costo, riesgo, calidad, interesados y comunicaciones.

Área PMBOKEjemplo en proyecto de software

CronogramaPlan de releases/sprints y ruta crítica
RiesgosDependencias externas, deuda técnica, rotación de equipo
CalidadEstándares de codificación, cobertura de pruebas, gates CI/CD
StakeholdersMapa de interesados y plan de comunicación por rol

Madurez de procesos: CMM y CMMI

NivelEstado del procesoQué caracteriza al equipo
1 - InicialAd-hoc / reactivoÉxitos dependen de personas, no del proceso
2 - GestionadoProyectos planificadosHay seguimiento de costo, tiempo y alcance
3 - DefinidoProceso organizacional estándarBuenas prácticas documentadas y reutilizables
4 - Cuantitativamente gestionadoMétricas y control estadísticoDecisiones por datos y variabilidad controlada
5 - OptimizadoMejora continua e innovaciónPrevención de defectos y aprendizaje sistemático

CMM fue el modelo inicial; CMMI evoluciona ese enfoque integrando múltiples disciplinas y una visión moderna de capacidad/madurez organizacional.
