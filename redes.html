Mascara de Subred

La mascara de subred determina que parte de la IP es la red y cual es el host. Los bits en 1 representan la porcion de red; los bits en 0 representan la porcion de host.

Operacion AND: IP AND Mascara = Direccion de Red

1.5 Ethernet y Switching

Direcciones MAC

Una direccion MAC tiene 48 bits (6 bytes), escritos en hexadecimal: AA:BB:CC:DD:EE:FF

Unicast
Bit menos significativo del primer byte = 0
Multicast
Bit menos significativo del primer byte = 1
Broadcast
FF:FF:FF:FF:FF:FF

Tabla CAM del Switch

El switch aprende direcciones MAC observando la MAC de origen de cada trama recibida y asociandola con el puerto de entrada.

  1. Trama llega al puerto → switch lee MAC origen
  2. Asocia MAC con el puerto en la tabla CAM
  3. Busca MAC destino en la tabla
  4. Si la encuentra → envia por ese puerto (unicast)
  5. Si no la encuentra → inunda (flood) por todos los puertos excepto el de origen

Dominios de Colision vs Broadcast

Dominio de Colision

Segmento donde dos tramas pueden colisionar. Un switch separa dominios de colision (cada puerto es uno). Un hub comparte un solo dominio.

Dominio de Broadcast

Segmento alcanzado por una trama broadcast. Un switch NO separa dominios de broadcast. Un router SI los separa.

Half-Duplex vs Full-Duplex

Half-DuplexFull-Duplex
DireccionUn sentido a la vezAmbos sentidos simultaneamente
ColisionesPosibles (usa CSMA/CD)No hay colisiones
EjemploHub, Walkie-talkieSwitch, Telefono

CSMA/CD

Carrier Sense Multiple Access / Collision Detection: antes de transmitir, el dispositivo escucha el medio. Si detecta una colision, envia una senal de jam y espera un tiempo aleatorio (backoff) antes de reintentar.

2.1 VLANs (Virtual LANs)

Una VLAN permite segmentar logicamente una red en dominios de broadcast independientes sin necesidad de hardware separado.

Tipos de VLANs

Puertos Access vs Trunk

Puerto Access

  • Pertenece a UNA sola VLAN
  • Conecta dispositivos finales (PC, impresora)
  • Trafico sin etiquetar

Puerto Trunk

  • Transporta MULTIPLES VLANs
  • Conecta switches entre si o switch-router
  • Usa etiquetas 802.1Q

Enrutamiento Inter-VLAN

Router-on-a-Stick

  • Un enlace trunk entre switch y router
  • Subinterfaces en el router (Gi0/0.10, Gi0/0.20)
  • Cada subinterfaz = una VLAN con su IP
  • Limitado por el ancho de banda del enlace

Switch Capa 3 (SVI)

  • Switch con capacidad de enrutamiento
  • Interfaz VLAN (SVI) con IP para cada VLAN
  • Mas rapido (enruta en hardware)
  • Requiere: ip routing

2.2 Spanning Tree Protocol (STP)

STP previene bucles en redes con enlaces redundantes bloqueando puertos selectivamente.

Problema: Tormentas de Broadcast

Sin STP, una trama broadcast circularia indefinidamente entre switches, saturando la red. STP crea una topologia libre de bucles.

Eleccion del Root Bridge

  1. Todos los switches envian BPDUs (Bridge Protocol Data Units)
  2. Se elige el switch con el Bridge ID mas bajo
  3. Bridge ID = Prioridad (4 bits, default 32768) + MAC
  4. Si la prioridad es igual, gana la MAC mas baja

2.4 Conceptos de Enrutamiento

Decision de Reenvio del Router

  1. Recibe paquete, lee IP destino
  2. Busca en la tabla de enrutamiento la coincidencia mas larga (longest prefix match)
  3. Si encuentra ruta, reenvia por la interfaz indicada
  4. Si no hay ruta, descarta (o envia al gateway por defecto si existe)

Metodologia de Subnetting

Pasos para Subnettear

1 Determinar cuantos bits de host se necesitan: 2^n - 2 >= hosts requeridos

2 Calcular la nueva mascara de subred

3 Determinar el tamano del bloque (block size) = 256 - valor del octeto de la mascara

4 Listar las subredes: 0, block_size, 2*block_size, ...

5 Para cada subred: red, primer host = red+1, ultimo host = broadcast-1, broadcast = siguiente_red - 1

Metodo del Numero Magico

El numero magico = 256 - el valor del octeto interesante de la mascara. Las subredes incrementan de ese valor.

Ejemplo: Mascara 255.255.255.192
Numero magico = 256 - 192 = 64
Subredes: .0, .64, .128, .192

Teoria VLSM

Que es VLSM?

VLSM (Variable Length Subnet Mask) permite dividir una red en subredes de diferentes tamanos, asignando exactamente el espacio necesario a cada subred. Esto es mas eficiente que FLSM donde todas las subredes son iguales.

Diferencia FLSM vs VLSM

FLSM

  • Todas las subredes del mismo tamano
  • Desperdicia direcciones
  • Mas simple de configurar

VLSM

  • Subredes de diferente tamano
  • Aprovecha mejor el espacio
  • Requiere planificacion ordenada

Reglas VLSM

1 Ordenar las subredes de mayor a menor (por cantidad de hosts)

2 Calcular para cada subred: encontrar la menor potencia de 2 que satisfaga hosts + 2 (red + broadcast)

3 Asignar bloques contiguos: cada subred inicia donde termino la anterior

4 Siguiente subred = broadcast de la anterior + 1

Introduccion a IPv6

Por que IPv6?

IPv4 tiene aproximadamente 4,300 millones de direcciones (2^32). Con el crecimiento de Internet y el IoT, estas direcciones se agotaron. IPv6 usa 128 bits, proporcionando 2^128 = 340 sextillones de direcciones.

Formato de Direccion IPv6

128 bits divididos en 8 grupos de 16 bits (hextetos), separados por dos puntos (:), escritos en hexadecimal.

2001:0DB8:0000:0000:0000:0000:0000:0001

Reglas de Compresion

  • Eliminar ceros a la izquierda de cada hexteto: 0DB8 → DB8, 0000 → 0
  • Usar :: para reemplazar UNO O MAS grupos consecutivos de solo ceros
  • :: solo se puede usar UNA VEZ en una direccion
2001:0DB8:0000:0000:0000:0000:0000:0001
→ 2001:DB8:0:0:0:0:0:1
→ 2001:DB8::1

EUI-64

EUI-64 genera automaticamente los ultimos 64 bits del identificador de interfaz a partir de la direccion MAC.

1 Tomar la MAC de 48 bits (ej: AA:BB:CC:DD:EE:FF)

2 Dividir en dos mitades: AA:BB:CC | DD:EE:FF

3 Insertar FF:FE en el medio: AA:BB:CC:FF:FE:DD:EE:FF

4 Invertir el 7mo bit (U/L bit) del primer byte

Autoconfig y NDP

SLAAC (Stateless Address Autoconfiguration)

El host genera su propia IPv6 usando el prefijo anunciado por el router (RA - Router Advertisement) y EUI-64 o un ID aleatorio. No requiere servidor DHCP.

DHCPv6

Stateless DHCPv6

El host usa SLAAC para la IP pero obtiene otros parametros (DNS) del servidor DHCPv6.

Stateful DHCPv6

El servidor DHCPv6 asigna la IP completa y todos los parametros, similar a DHCPv4.

ICMPv6 y NDP (Neighbor Discovery Protocol)

NDP reemplaza a ARP y agrega funcionalidades adicionales:

MensajeTipoFuncion
Router Solicitation (RS)133Host pregunta por routers en el enlace
Router Advertisement (RA)134Router anuncia prefijo y parametros
Neighbor Solicitation (NS)135Resolucion de direccion (como ARP request)
Neighbor Advertisement (NA)136Respuesta con MAC (como ARP reply)
Redirect137Indica mejor next-hop al host

Mecanismos de Transicion

Dual Stack

El dispositivo ejecuta IPv4 e IPv6 simultaneamente. Es la estrategia preferida.

Tunneling

Paquetes IPv6 se encapsulan dentro de paquetes IPv4 para cruzar redes IPv4 (6to4, ISATAP, GRE).

NAT64

Traduce entre IPv6 e IPv4. Permite a hosts IPv6 comunicarse con hosts IPv4.
