¿Qué es una red?

Una red es un conjunto de dispositivos conectados para intercambiar información y compartir recursos.

Componentes

  • Hosts (PC, laptop, móvil, servidor)
  • Medios (cable, fibra, Wi-Fi)
  • Equipos intermedios (switch, router)

Objetivos

  • Comunicación
  • Compartir recursos
  • Acceso a servicios

Escala

  • LAN: red local
  • MAN: metropolitana
  • WAN: larga distancia / Internet

Fundamentos clave

Ancho de banda vs latencia

Ancho de banda: cantidad de datos por segundo (Mbps/Gbps). Latencia: tiempo que tarda un paquete en ir y volver (ms).

Paquete y trama

Los datos se dividen en unidades. En capa de red se habla de paquetes; en capa de enlace, de tramas.

Unicast, Broadcast y Multicast

Uno a uno, uno a todos, uno a muchos (grupo).

Cliente-servidor

Cliente solicita, servidor responde. Ejemplo: navegador ↔ servidor web.

Modelo TCP/IP (4 capas)

Aplicación

HTTP, HTTPS, DNS, DHCP, SMTP, FTP.

Transporte

TCP (confiable), UDP (rápido, sin confirmación).

Internet

IP, ICMP, ARP para direccionamiento y ruteo.

Direccionamiento IP básico

IPv4

32 bits, formato decimal con puntos: 192.168.10.25.

  • Clase privada A: 10.0.0.0/8
  • Clase privada B: 172.16.0.0/12
  • Clase privada C: 192.168.0.0/16

IPv6

128 bits, mayor espacio de direcciones: 2001:db8::1.

  • Sin NAT obligatorio
  • Autoconfiguración más robusta
  • Diseñado para crecimiento de Internet

Conceptos indispensables

Máscara

Define qué parte es red y qué parte es host. Ejemplo: /24 = 255.255.255.0.

Gateway

Puerta de salida a otras redes, normalmente la interfaz del router.

DNS

Traduce nombres de dominio a direcciones IP.

Subredes (nivel inicial)

Subredar es dividir una red grande en redes más pequeñas para ordenar, mejorar seguridad y optimizar tráfico.

PrefijoMáscaraHosts útilesUso común
/24255.255.255.0254LAN pequeña/mediana
/25255.255.255.128126Segmentar dos áreas
/26255.255.255.19262Oficinas pequeñas
/27255.255.255.22430VLAN departamental
/30255.255.255.2522Enlace punto a punto

Ejemplo rápido

Red base: 192.168.1.0/24. Si la divides en /26 obtienes 4 subredes:

  • 192.168.1.0/26 (hosts .1 a .62, broadcast .63)
  • 192.168.1.64/26 (hosts .65 a .126, broadcast .127)
  • 192.168.1.128/26 (hosts .129 a .190, broadcast .191)
  • 192.168.1.192/26 (hosts .193 a .254, broadcast .255)

Equipos de red esenciales

Switch (Capa 2)

Conecta equipos dentro de la LAN y reenvía tramas por dirección MAC.

Router (Capa 3)

Conecta redes distintas y decide la mejor ruta para paquetes IP.

Access Point

Conecta dispositivos inalámbricos (Wi-Fi) a la red cableada.

Otros componentes

Firewall

Filtra tráfico según reglas de seguridad.

Modem/ONT

Interfaz entre red local y proveedor de Internet.

Servidor DHCP

Asigna IP automáticamente a clientes.

Servidor DNS

Resuelve nombres de dominio a direcciones IP.

Redundancia de gateway virtual (FHRP)

Cuando hay dos o más routers para salida, se usan protocolos de redundancia de primer salto para mantener una puerta de enlace virtual activa aun si un router falla.

Protocolos comunes

  • HSRP (Cisco): un router activo y uno standby.
  • VRRP (estándar): router master y backup.
  • GLBP (Cisco): además balancea carga entre routers.

Pregunta tipo examen

En un escenario de redundancia de routers, ¿qué permite usar una IP de gateway virtual?

Respuesta esperada: un protocolo FHRP (por ejemplo HSRP o VRRP).
